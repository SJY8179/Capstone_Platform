Claude Code 작업 프롬프트 (붙여넣기용)

역할: 프런트엔드 리팩터/버그 헌터
목표:

프로젝트 생성 성공 시 모달을 닫는다.

목록은 즉시 갱신된다(낙관적 추가 or invalidate).

우측 상단 “새 프로젝트” 버튼은 항상 다시 동작한다(비활성/무응답 금지).

“계속 생성하기” 체크박스는 제거한다.

해야 할 일

코드 탐색

frontend/src/components/Projects/CreateProjectModal.(tsx|jsx)

frontend/src/pages/Projects/ProjectManagement.(tsx|jsx) 또는 ProjectsPage

frontend/src/api/projects.(ts|js) (createProject 사용)

CreateProjectModal 수정

“계속 생성하기” 관련 import/상태/UI 전부 삭제.

제출 흐름:

try {
  setIsSubmitting(true);
  const created = await createProject(payload);
  toast.success('프로젝트가 생성되었습니다.');
  onCreateSuccess?.(created);   // 리스트 갱신
  onClose();                    // ✅ 성공 시 모달 닫기
  resetForm();                  // 닫기 직전/직후 폼 초기화(잔상 방지)
} catch (e) {
  toast.error('생성에 실패했습니다.');
} finally {
  setIsSubmitting(false);       // 제출 버튼만 비활성 → 복구
}


모달 언마운트/닫힘 시 isSubmitting=false 보장하는 cleanup 추가.

ESC/바깥 클릭/“취소”로 정상 종료 유지.

다시 “새 프로젝트” 클릭 → 모달이 정상 재오픈(버튼 안 죽음)

실패 플로우

500 모킹 시 토스트 노출, 모달 유지, 제출 버튼만 재활성화

접근성/잔상

모달 닫힌 뒤 화면 클릭 가능(백드롭 pointer-events 잔류 없음)

수용 기준 (Acceptance Criteria)

(A1) 생성 성공 시 모달은 항상 닫힌다.

(A2) 목록은 즉시 갱신된다(낙관적 업데이트 또는 invalidate 후 재조회).

(A3) “새 프로젝트” 버튼은 항상 재동작하며 비활성/무응답 상태로 남지 않는다.

(A4) 체크박스 없이도 위 동작을 만족한다.

(A5) 실패 시에도 트리거 버튼은 영향받지 않는다(모달 내부 제출 버튼만 상태 관리).

// CreateProjectModal.tsx (핵심)
const handleSubmit = async (e) => {
  e.preventDefault();
  try {
    setIsSubmitting(true);
    const created = await createProject(form);
    toast.success('프로젝트가 생성되었습니다.');
    onCreateSuccess?.(created); // 목록 갱신
    onClose();                  // ✅ 성공 시 닫기
    resetForm();
  } catch (err) {
    toast.error('생성에 실패했습니다.');
  } finally {
    setIsSubmitting(false);
  }
};

// ProjectManagement.tsx (핵심)
<Button onClick={handleOpenCreateModal}>
  <Plus className="h-4 w-4 mr-2" /> 새 프로젝트
</Button>
<CreateProjectModal
  isOpen={isCreateModalOpen}
  onClose={handleCloseCreateModal}
  onCreateSuccess={handleCreateSuccess}
/>
