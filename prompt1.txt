지금까지 구현된 기능을 정리해서 보고서를 작성해 줘.
출력은 도메인 설계서 + REST API 설계서 + 화면 설계서 형식을 결합한 마크다운으로 작성해.

포함할 내용

문서 정보: 프로젝트명, 작성자, 작성일, 버전

프로젝트 개요: 목적, 범위, 이해관계자

구현된 기능 요약

프론트엔드 (React/Vite + shadcn/Radix): 버전 고정, Tailwind 설정, Vite 프록시, JWT 인터셉터, LoginDemo 페이지

백엔드 (Spring Boot + JPA + MariaDB): JWT 인증, CORS 설정, REST API (로그인/로그아웃/토큰갱신/헬스체크)

CI/개발환경: Node 버전 lock-in, package-lock.json 고정, verify-radix 스크립트

도메인 객체 및 관계도: 현재까지 확정된 엔티티와 관계

REST API 목록: /auth/login, /auth/logout, /auth/refresh, /api/health 등 현재 동작하는 API 엔드포인트

화면 설계: LoginDemo 포함, Header/Sidebar/Main Content/ Footer 구조

비즈니스 규칙: JWT 인증 필수, 특정 권한 제약, import 경로 통일 등

가정 및 제약조건: Node 18 고정, 프론트 포트 5194, 백엔드 포트 8080

향후 확장 계획: CI/CD 파이프라인에 verify-radix 추가, 사용자 관리 페이지, 일정 관리 기능 확장

출력 형식

예시 도메인 설계서, REST API 설계서, 화면 설계서의 마크다운 구조를 따를 것
